# VybeCode Privacy Policy

Effective date: June 15, 2026

This Privacy Policy describes privacy practices for the public `prismworks-ai/vybecode` release repository and the VybeCode desktop app binaries published here.

## 1. Public GitHub Repository

Information you submit to this repository through issues, comments, attachments, discussions, or pull requests may be public and processed by GitHub under GitHub's own terms and policies.

Do not submit secrets, API keys, tokens, private repository content, private logs, customer data, or confidential information in public issues.

## 2. Desktop App Data

VybeCode is a desktop application that can interact with local files, local tools, GitHub, AI model providers, CLI agents, and self-hosted endpoints according to how you configure it.

Depending on your use, the app may process:

- project paths and workspace metadata
- local settings and preferences
- model provider configuration
- chat prompts and responses
- CLI agent activity
- logs or diagnostics that you choose to share

Provider API keys and local configuration should remain on your device unless you intentionally provide them to a third-party service or include them in a support report. Do not include secrets in GitHub Issues.

## 3. Third-Party Services

When you configure VybeCode to use GitHub, model providers, CLI agents, hosted APIs, or self-hosted endpoints, those services may receive data needed to perform the requested action. Their privacy practices are governed by their own policies.

You are responsible for reviewing third-party provider terms and privacy practices before connecting them to VybeCode.

## 4. Support and Diagnostics

If you submit an issue or support report, PrismWorks AI may use the information you provide to investigate, reproduce, prioritize, fix, and document the reported issue.

Only share logs and screenshots after removing secrets and confidential information.

## 5. Data Retention

Public GitHub issue content remains available according to GitHub repository history, issue retention, and deletion behavior. PrismWorks AI may retain issue information for product support, security, compliance, and release management purposes.

## 6. Changes

PrismWorks AI may update this policy as VybeCode and its release process evolve. Updated versions will be published in this repository.
