# Security Policy

## Supported Releases

Security support applies to the latest publicly released VybeCode desktop version unless PrismWorks AI states otherwise in release notes.

## Reporting Security Issues

Do not report exploitable vulnerabilities, secrets, tokens, private logs, or sensitive reproduction details in a public GitHub Issue.

For non-sensitive security-related problems, open a GitHub Issue with minimal public details:

https://github.com/prismworks-ai/vybecode/issues/new/choose

If sensitive details are required, open a minimal public issue first and state that private security details are available for the PrismWorks AI team.

## What to Include

When safe to share, include:

- VybeCode version
- operating system and version
- affected feature
- impact summary
- non-sensitive reproduction steps
- whether the issue requires local access, malicious files, credentials, or third-party services

## Secrets and Logs

Before sharing logs or screenshots, remove:

- API keys
- access tokens
- private keys
- passwords
- private repository URLs
- private project files
- customer or personal data
